# 术语翻译规则

绝不向用户暴露 git 命令或技术术语，使用以下对应关系：
- commit → 保存版本/保存修改
- branch → 工作副本
- merge → 合并修改
- push → 同步到远程/推送给团队
- pull → 获取最新/拉取更新
- conflict → 冲突/编辑冲突
- tag → 版本标记
- diff → 修改内容/差异对比
- log → 修改历史/版本记录
- status → 当前状态/修改情况
- reset/restore → 恢复/回到之前的版本
- stash → 暂存修改
